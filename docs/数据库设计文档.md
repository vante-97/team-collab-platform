# 团队协作平台 - 数据库设计文档

## 1. 概述

- **数据库类型**：SQLite 3
- **ORM 框架**：SQLAlchemy
- **数据库文件**：`backend/data.db`（首次运行自动创建）

### 关系说明

| 关系 | 类型 | 说明 |
|------|------|------|
| User → Project | 1:N | 一个用户可创建多个项目 |
| Project → Task | 1:N | 一个项目包含多个任务 |
| User → Task | 1:N | 一个用户可被分配多个任务 |
| Project → TeamMember | 1:N | 一个项目有多个成员 |
| User → TeamMember | 1:N | 一个用户可加入多个项目 |
| Project → ProjectFile | 1:N | 一个项目包含多个文件 |
| User → ProjectFile | 1:N | 一个用户可上传多个文件 |

## 3. 表结构详细设计

### 3.1 users（用户表）

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| `id` | INTEGER | PK, AUTO_INCREMENT | 用户唯一标识 |
| `username` | VARCHAR(80) | UNIQUE, NOT NULL | 用户名 |
| `email` | VARCHAR(120) | UNIQUE, NOT NULL | 邮箱地址 |
| `password_hash` | VARCHAR(256) | NOT NULL | 密码哈希（werkzeug） |
| `created_at` | DATETIME | DEFAULT UTC_NOW | 注册时间 |

### 3.2 projects（项目表）

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| `id` | INTEGER | PK, AUTO_INCREMENT | 项目唯一标识 |
| `name` | VARCHAR(200) | NOT NULL | 项目名称 |
| `description` | TEXT | DEFAULT "" | 项目描述 |
| `status` | VARCHAR(20) | DEFAULT "planning" | 项目状态 |
| `owner_id` | INTEGER | FK → users.id, NULLABLE | 创建者 |
| `created_at` | DATETIME | DEFAULT UTC_NOW | 创建时间 |
| `updated_at` | DATETIME | DEFAULT UTC_NOW, ON UPDATE | 更新时间 |

**status 枚举值**：

| 值 | 含义 |
|----|------|
| `planning` | 规划中 |
| `active` | 进行中 |
| `completed` | 已完成 |
| `archived` | 已归档 |

### 3.3 tasks（任务表）

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| `id` | INTEGER | PK, AUTO_INCREMENT | 任务唯一标识 |
| `title` | VARCHAR(300) | NOT NULL | 任务标题 |
| `description` | TEXT | DEFAULT "" | 任务描述 |
| `status` | VARCHAR(20) | DEFAULT "todo" | 任务状态 |
| `priority` | VARCHAR(10) | DEFAULT "medium" | 优先级 |
| `assignee_id` | INTEGER | FK → users.id, NULLABLE | 负责人 |
| `project_id` | INTEGER | FK → projects.id, NOT NULL | 所属项目 |
| `created_at` | DATETIME | DEFAULT UTC_NOW | 创建时间 |
| `updated_at` | DATETIME | DEFAULT UTC_NOW, ON UPDATE | 更新时间 |

**status 枚举值**：

| 值 | 含义 |
|----|------|
| `todo` | 待办 |
| `in_progress` | 进行中 |
| `done` | 已完成 |

**priority 枚举值**：

| 值 | 含义 |
|----|------|
| `low` | 低优先级 |
| `medium` | 中优先级 |
| `high` | 高优先级 |
| `urgent` | 紧急 |

### 3.4 team_members（团队成员表）

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| `id` | INTEGER | PK, AUTO_INCREMENT | 记录唯一标识 |
| `user_id` | INTEGER | FK → users.id, NOT NULL | 用户 |
| `project_id` | INTEGER | FK → projects.id, NOT NULL | 项目 |
| `role` | VARCHAR(20) | DEFAULT "member" | 角色 |
| `joined_at` | DATETIME | DEFAULT UTC_NOW | 加入时间 |

**role 枚举值**：

| 值 | 含义 | 权限 |
|----|------|------|
| `owner` | 创建者 | 全部权限 |
| `admin` | 管理员 | 管理成员、编辑项目 |
| `member` | 成员 | 创建任务、上传文件 |
| `viewer` | 观察者 | 仅查看 |

### 3.5 project_files（项目文件表）

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| `id` | INTEGER | PK, AUTO_INCREMENT | 文件唯一标识 |
| `filename` | VARCHAR(500) | NOT NULL | 存储文件名（UUID） |
| `original_name` | VARCHAR(500) | NOT NULL | 原始文件名 |
| `file_size` | INTEGER | DEFAULT 0 | 文件大小（字节） |
| `file_type` | VARCHAR(50) | DEFAULT "" | 文件类型分类 |
| `uploader_id` | INTEGER | FK → users.id, NOT NULL | 上传者 |
| `project_id` | INTEGER | FK → projects.id, NOT NULL | 所属项目 |
| `created_at` | DATETIME | DEFAULT UTC_NOW | 上传时间 |

**file_type 分类**：

| 值 | 对应扩展名 |
|----|-----------|
| `pdf` | .pdf |
| `image` | .png, .jpg, .jpeg, .gif, .webp, .svg, .bmp |
| `doc` | .doc, .docx, .xls, .xlsx, .ppt, .pptx |
| `text` | .txt, .md, .json, .xml, .csv, .log |
| `other` | 其他所有格式 |

## 4. 索引设计

当前使用 SQLAlchemy 默认索引：

| 表 | 索引字段 | 类型 | 说明 |
|----|---------|------|------|
| users | `username` | UNIQUE | 用户名唯一索引 |
| users | `email` | UNIQUE | 邮箱唯一索引 |
| 所有表 | `id` | PRIMARY KEY | 主键索引 |
| 所有表 | FK 字段 | FOREIGN KEY | 外键自动索引 |

## 5. 数据初始化

系统首次启动时，`init_db()` 函数会自动执行：

1. **创建表结构**：`db.create_all()`
2. **插入测试用户**（当 users 表为空时）：
   - `testuser1` / `123456` / `testuser1@team.com`
   - `testuser2` / `123456` / `testuser2@team.com`
   - `admin` / `admin123` / `admin@team.com`
3. **插入示例项目**（当 projects 表为空时）：
   - "测试样例项目"（进行中）
   - "前端重构"（规划中）
   - "API 文档编写"（已完成）

## 6. 级联删除策略

| 删除操作 | 级联效果 |
|---------|---------|
| 删除项目 | 自动删除该项目下所有任务、成员、文件 |
| 删除用户 | 不会级联删除（外键 nullable） |
| 删除文件记录 | 同时删除物理文件 |

---

## 7. 扩展建议

如需迁移到生产级数据库（如 PostgreSQL），只需修改 `SQLALCHEMY_DATABASE_URI` 配置：

```python
app.config["SQLALCHEMY_DATABASE_URI"] = "postgresql://user:pass@localhost/dbname"
```

SQLAlchemy ORM 层保证了代码无需其他修改。
