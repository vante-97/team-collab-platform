# 个人总结报告

> 项目名称：团队协作平台 (Team Collab Platform)  
> 负责人：胡超群  
> 负责模块：前端页面开发、UI 设计、权限控制、部署配置、文档完善

---

## 一、项目概述

我们这个项目是用 Next.js 14 + TypeScript + Tailwind CSS 搭的一个团队协作平台，可以管理项目、分配任务、上传文件什么的。我在组里主要负责前端这块，从登录注册页到仪表盘、文件管理，再到部署上线和文档，基本上前端相关的活儿都是我在搞。

## 二、主要工作

前端页面这块，我最先做的是登录和注册页面，接上了后端的 JWT 认证。然后做了文件上传管理和数据统计仪表盘两个大模块，后面又优化了仪表盘的布局、按钮样式和权限标签。

权限控制是我花时间比较多的部分。一开始所有登录用户都能看到别人的项目，不太安全。我加了项目成员权限检查，把项目列表和仪表盘数据都限定为当前用户能访问的范围。还修了一个任务创建的 bug，之前创建失败不给错误提示，我改了 fetchApi 让 4xx 响应不再直接抛异常，用户能看到反馈。

UI 方面，暗色模式下文字对比度太低，加上动画半透明效果，有些字根本看不清，我调了颜色深度和字号。另外还修了一些 ESLint 警告，把没用到的变量和 Hook 依赖清理了。

部署这边，我配了 Vercel 前端的部署配置，遇到了 TypeScript 和 ESLint 检查导致构建失败的问题，最后跳过构建检查才成功上线。README 文档也是我完善和更新的。

## 三、遇到的困难与解决

最大的坑是 Vercel 构建一直失败，TS 类型检查和 ESLint 报了一堆错。最后只能先跳过检查让它跑起来。暗色模式那个也调了好久，Tailwind 默认的暗色文字颜色太淡了，配上半透明动画简直灾难。

## 四、收获与体会

做完这个项目，我对前端开发的全流程理解更深了。以前觉得前端就是画页面，现在知道还要考虑权限边界、状态管理和不同环境下的构建差异。AI 辅助编程写组件确实快，但逻辑和细节还是得自己把关。整体来说收获挺大的，从零到上线走了一遍。
