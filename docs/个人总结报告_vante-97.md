# 个人总结报告

> 项目名称：团队协作平台 (Team Collab Platform)  
> 负责人：章丽霞  
> 负责模块：后端 API 开发、数据库设计、安全加固、CORS 跨域、生产部署

---

## 一、项目概述

我们这个项目是一个前后端分离的团队协作平台，前端用 Next.js，后端用 Flask。我主要负责后端，从数据库设计、API 开发到安全加固和 Railway 部署，基本后端的事儿都是我在搞。

## 二、主要工作

最开始我搭了整个项目的初始仓库，然后实现了用户注册 API，用了 JWT 和 bcrypt 做认证。接着写了项目管理的 5 个接口，又做了任务看板和团队协作模块。邀请系统这块我做了收件箱机制，成员可以通过接受或拒绝来加入项目，不是直接拉人。

修 bug 也花了不少时间。数据库初始化脚本有 bug，会自动给所有项目加示例成员；项目创建者没有被自动设成 owner，导致建了项目却创建不了任务；项目详情的邀请流程和成员显示也有问题，这些我都修了。

安全方面，JWT 密钥之前是硬编码在代码里的，我改成了从环境变量读取，Debug 模式也加了开关。文件上传加了类型白名单和安全文件名过滤，密码提示统一成至少 8 位。

CORS 跨域是这次最折腾的部分。前端请求带 Authorization 头会被拦，我搞了 CORS 白名单、动态匹配 railway.app 子域名、全局 OPTIONS 预检处理，前前后后试了好多方案才搞定。部署到 Railway 的时候也踩了不少坑——bcrypt 依赖没写进 requirements.txt、路由蓝图没注册导致 502、数据库每次部署都会重置，全都是一步步排查解决的。

## 三、遇到的困难与解决

CORS 跨域问题搞了最久。从加 Authorization 头支持，到处理预检请求被 301 重定向，再到禁用 Flask-CORS 自动 OPTIONS 避免 404 覆盖响应，折腾了好几轮才彻底弄好。还有 Railway 部署 502 崩溃，最后发现是任务、成员、文件这些路由蓝图没注册，加上就好了。

## 四、收获与体会

这次项目让我对后端开发有了比较完整的认识，从写接口到上线部署全流程都走了一遍。最大的体会是"能跑"和"能上线"差别太大了，生产环境要考虑依赖缺失、环境变量、CORS、数据库持久化一堆问题。AI 写 CRUD 确实快，但安全问题、跨域配置这些还是得自己理解原理才行。
