## 一、项目背景与目标

本项目旨在开发一个功能完善的团队协作管理平台，帮助团队成员高效地进行项目管理、任务分配、文件共享和数据统计。平台采用前后端分离架构，前端使用 Next.js 14 + TypeScript + Tailwind CSS，后端使用 Python Flask + SQLAlchemy + JWT 认证，数据库使用 SQLite。

## 二、完成的工作

### 2.1 功能开发

完成了 **10 个前端页面**和 **25 个后端 API 接口**的开发，涵盖以下功能模块：

1. **用户认证系统**：注册、登录、JWT 双 Token 认证、Token 自动刷新、路由保护
2. **仪表盘首页**：快捷入口、实时统计、最近项目和任务展示
3. **项目管理**：CRUD 操作、状态流转（规划中→进行中→已完成→已归档）、项目详情四 Tab（概览/任务/成员/文件）
4. **任务看板**：三列布局（待办/进行中/已完成）、任务拖拽、优先级管理
5. **团队协作**：成员邀请/接受/拒绝、角色管理（owner/admin/member/viewer）、收件箱
6. **文件管理**：上传/下载/删除、文件类型白名单（30+）、大小限制（50MB）、安全文件名过滤
7. **数据统计**：项目/任务/成员/文件统计、任务状态分布、优先级分布
8. **UI 设计系统**：暗色/亮色主题切换、玻璃态卡片、统一动画系统、响应式布局

## 四、遇到的困难与解决方案

### 4.1 CORS 跨域问题

**困难：** 前端部署到 EdgeOne 后，带 Authorization 头的 API 请求被浏览器拦截，返回 CORS 错误。

**解决过程：**
1. 首先确认后端 CORS 配置是否正确（Flask-CORS 的 `allow_headers` 配置）
2. 使用 PowerShell 模拟 OPTIONS 预检请求，确认响应头正确
3. 发现 CORS 白名单中缺少 EdgeOne 域名，添加后解决

### 4.2 Railway 部署依赖缺失

**困难：** 后端部署到 Railway 后启动失败，报错 `No module named 'bcrypt'`。

**解决：** 在 `requirements.txt` 中添加 `bcrypt==4.2.0`，提交推送后 Railway 自动重新部署成功。

### 4.3 项目管理页面 API 请求失败（未完全解决）

**困难：** 登录注册正常，但项目管理页面的 API 请求返回 "Failed to fetch"。Console 手动 fetch 成功返回 200，但页面自动请求失败。

**排查进展：** 已确认后端正常运行、CORS 配置正确、JWT Token 有效。可能是 EdgeOne CDN 对带 Authorization 头的请求有特殊处理。待进一步排查。
